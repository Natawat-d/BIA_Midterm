DSS & Simon's decision-making process

DSS: "interactive computer-based systems that help decision makers use data and models to solve unstructured problems" (Gorry & Scott-Morton, 1971). BI is an evolution of DSS concepts.

Components of a DSS

  • Data management — internal/external data (ERP/POS, legacy, web)
  • Model management — models + external models
  • Knowledge-based subsystems
  • User interface

Simon's four phases

Intelligence — define the problem or opportunity.
Design — build a model; identify possible solutions.
Choice — compare, choose and recommend a solution.
Implementation — put the solution to work (+ monitoring).

BI framework & the three analytics types

BI (broad): an umbrella term combining architectures, tools, databases, analytical tools, applications and methodologies. Objective: easy access to data so managers can analyse and act.

Four components of a BI system

  • Business analytics — tools to manipulate/mine/analyse
  • Business Performance Management (BPM) — monitor & analyse performance
  • User interface — e.g. dashboard

Data preprocessing — the "art & science"

Real-world data is dirty, misaligned, complex, inaccurate. Four tasks (with feedback loops) make it analytics-ready:

Consolidation→Cleaning→ Transformation→Reduction
TaskMethods
ConsolidationAccess, select, integrate (SQL, web services, ontology mapping)
CleaningImpute missing (mean/median/mode), reduce noise/outliers, remove errors
TransformationNormalize (0–1), discretize/aggregate (binning), construct attributes
ReductionReduce attributes (PCA, correlation), records (sampling), balance skew (over/under-sample)

Statistical modeling & regression

Statistics = techniques to characterise & interpret data. Descriptive (describe data) vs Inferential (conclude about a population from a sample).

AspectMeasures
Central tendencyMean, median, mode
DispersionRange, variance, std dev, quartiles, box-plot
ShapeHistogram, skewness (asymmetry), kurtosis (peakedness)

Regression — the most widely used technique

  • Simple: y = β₀ + β₁x · Multiple: y = β₀ + β₁x₁ + … + βₙxₙ
  • Fit by ordinary least squares (minimise squared errors)
  • Used for explanation (hypothesis testing) and prediction (forecasting)
  • Quality: R², p-values, errors MSE / MAD / RMSE
Five assumptions: linearity · independence · normality · constant variance · no multicollinearity.

Logistic regression

A statistics-based classification algorithm (supervised). Target is a binary variable via the sigmoid f(y)=1/(1+e−(β₀+β₁x)).

Ch.5 Data Mining: Process, Methods & Algorithms (Predictive Analytics I)

What is data mining?

Data mining = "the nontrivial process of identifying valid, novel, potentially useful, and ultimately understandable patterns in data" (Fayyad et al., 1996).

A.k.a. knowledge extraction/discovery, pattern analysis, information harvesting. It is a blend of disciplines: statistics, machine learning, AI, databases, visualization. The source is often a data warehouse (not always). Data mining ≠ statistics — DM is pattern-discovery on large data, often by an end user.

Types of patterns it extracts

Association·Prediction·Cluster (segmentation)·Sequential / time-series

A taxonomy of data-mining tasks

TaskMethodAlgorithmsLearning
PredictionClassificationDecision trees, ANN, SVM, kNN, Naïve BayesSupervised
RegressionLinear/nonlinear regression, ANN, regression trees, SVMSupervised
Time seriesExponential smoothing, ARIMA, autoregressiveSupervised
AssociationMarket-basketApriori, Eclat, FP-GrowthUnsupervised
Link / sequenceApriori, graph-based matchingUnsupervised
SegmentationClusteringK-means, Expectation-MaximizationUnsupervised
Outlier analysisK-means, EMUnsupervised
The two key contrasts (exam!): Classification vs regression — categorical vs numeric output. Classification vs clustering — supervised (has target) vs unsupervised (no target).

Standard processes — CRISP-DM, SEMMA, KDD

CRISP-DM (the most popular) has six phases — the first three eat ~85% of project time:

Business Understanding
Data Understanding
Data Preparation (steps 1–3 ≈ 85% of effort)
Model Building
Testing & Evaluation
Deployment

Highly iterative/experimental. Alternatives: SEMMA (SAS: Sample · Explore · Modify · Model · Assess) and KDD (Knowledge Discovery in Databases).

Classification — methods & accuracy

Most-used DM method: supervised, learns from past data to classify new data; output is categorical. Assess by accuracy, speed, robustness, scalability, interpretability.

Confusion matrix

Actual PositiveActual Negative
Predicted PositiveTPFP
Predicted NegativeFNTN
  • Accuracy = (TP+TN) / (TP+TN+FP+FN)
  • Precision = TP / (TP+FP) · Recall / TPR = TP / (TP+FN)
  • True Negative Rate = TN / (TN+FP)

Estimation methods

  • Simple split / holdout — train ~⅔ (70%), test ⅓ (30%); for NN: 60/20/20 train/validation/test
  • k-fold cross-validation — k train/test rounds (rotation estimation)
  • Leave-one-out (k = n), bootstrapping (sample with replacement), jackknifing
  • AUC / ROC curve — 0.5 = chance, 1.0 = perfect; robust to skewed classes

Decision trees

Divide-and-conquer: recursively split the training set until each leaf is one class. Algorithms differ on splitting, stopping, and pruning (pre- vs post-). Popular: ID3, C4.5, C5, CART, CHAID, M5.

Ensembles combine many models for more robust prediction — bagging (random forest), boosting, information fusion.

Clustering & association rules (unsupervised)

Cluster analysis

Finds natural groupings; no target variable; in marketing = segmentation. Methods: k-means / k-modes (statistical), SOM & ART (neural), fuzzy c-means, GA.

k-Means algorithm:

Choose k; randomly place k cluster centers.
Assign each point to the nearest center.
Re-compute centers; repeat until assignments are stable.

Association rule mining (market-basket)

Finds affinities between items — the famous "diapers & beer". Rule X → Y [S%, C%]:

  • Support (S) — how often X and Y appear together
  • Confidence (C) — how often Y appears when X does
  • Algorithms: Apriori (bottom-up, frequent itemsets), Eclat, FP-Growth
Myths vs reality: DM is not instant crystal-ball prediction — it is a deliberate multi-step process; it does not require a dedicated database or advanced degrees, and is viable for firms of any size.

Ch.6 Text, Web & Social Media Analytics (Predictive Analytics II)

Text analytics vs text mining vs data mining

  • Text Analytics = Information Retrieval + Information Extraction + Data Mining + Web Mining  (or simply IR + Text Mining)
  • Text mining = semi-automated extraction of knowledge from unstructured data
  • Data mining vs text mining: both find novel patterns; the difference is structured (databases) vs unstructured (documents) data. ~85–90% of corporate data is unstructured.
To mine text you must first impose structure on it, then mine the structured result.

Application areas: information extraction · topic tracking · summarization · categorization · clustering · concept linking · question answering.

Terminology & NLP

Key terms: corpus/corpora · terms · concepts · stemming · stop words · synonyms/polysemes · tokenizing · term-by-document matrix · SVD (latent semantic indexing).

NLP (subfield of AI & computational linguistics) moves from the old bag-of-words to "understanding" language. Challenges: POS tagging, text segmentation, word-sense disambiguation, syntactic ambiguity. Resources: WordNet, SentiWordNet.

The three-step text-mining process

1 · Establish the Corpus
collect & digitize unstructured data→ 2 · Create the TDM
term-document matrix = structure→ 3 · Extract Knowledge
classify / cluster / associate

The TDM is a sparse matrix; cells hold term frequencies (counts, binary, log, or TF-IDF). Reduce dimensionality with SVD (similar to PCA).

Topic modeling

Unsupervised discovery of themes: simple frequency clustering, LSI/LSA (TF-IDF + SVD), and the most popular — LDA (Latent Dirichlet Allocation). Assumes each document = mix of topics, each topic = collection of words.

Sentiment analysis

Answers "what do people feel about a topic?" Especially valuable in CRM. Applications: Voice of the Customer / Market / Employee, brand management, financial markets, politics, government intelligence.

Four-step process

Sentiment detection — objectivity: fact vs opinion (S–O polarity)
N-P polarity classification — negative vs positive
Target identification — what the sentiment is about
Collection & aggregation — word → statement → paragraph → document

Web mining & social media analytics

Web mining = discovering relationships from web data. Three branches:

Content mining

Page text
  • Search engines, IR, sentiment

Structure mining

URL links
  • PageRank, HITS (hubs & authorities), graph mining

Usage mining

Clicks & sessions
  • Clickstream analysis, web analytics

Web analytics metrics: site usability (page views, time on site), traffic sources, visitor profiles, conversion statistics.

Social Network Analysis (SNA): social structure of individuals linked to each other (graph theory + sociology). Metrics: centrality, density, betweenness/bridge, cliques, clustering coefficient. Social media analytics turns Web 2.0 content into competitive insight; engagement levels run Inactives → Spectators → Joiners → Critics → Creators.

Ch.9 The Landscape of Software Tools & Languages

Why tools matter — the toolbox approach

Analytics/data-science tooling is evolving fast. Some tools excel at preprocessing, others at model-building, others at ease of use. There is no single tool best at everything — pursue a toolbox approach. Tool-related success factors: rapid prototyping, cost of ownership, flexibility/expandability (open architecture), ease of deployment, security.

Quick self-test prompts

Contrast classification, regression and clustering.
Classification → categorical output, supervised. Regression → numeric output, supervised. Clustering → natural groupings, unsupervised (no target). Classification & clustering both assign items to groups, but only classification learns from labelled past data.
State the CRISP-DM phases and where time goes.
Business Understanding → Data Understanding → Data Preparation → Model Building → Testing & Evaluation → Deployment. The first three (especially data prep) consume ~85% of project effort.
Define support and confidence for X → Y.
Support = P(X and Y) — how often they co-occur. Confidence = P(Y | X) — how often Y appears given X. Apriori uses a bottom-up approach to find frequent itemsets above a minimum support.
Give the 3-step text-mining process.
1) Establish the corpus (collect/digitize). 2) Create the term-document matrix (impose structure; TF-IDF, reduce with SVD). 3) Extract knowledge (classification, clustering, association, trend).
Why "no single best analytics tool"?
Tools differ across free/paid, open/closed, local/server, visual/syntactic and excel at different stages (prep vs modeling vs deployment). Best practice is a toolbox — e.g. KNIME/Orange for visual workflows, R/Python for flexible coding, Tableau/Power BI for dashboards.
