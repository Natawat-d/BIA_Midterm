Ch.1 · Overview of BI & Analytics

Key idea · BI is the modern evolution of DSS; its job is to turn data into managerial action, and every method is descriptive, predictive or prescriptive.

DSS — interactive computer system that uses data + models to solve semi/unstructured problems (Gorry & Scott-Morton 1971). 4 components: data mgmt · model mgmt · knowledge subsystem · UI. Usage: couple human judgment + computing to raise decision quality.

Simon's 4 phases — Intelligence (find problem) → Design (build model/options) → Choice (pick) → Implementation (act+monitor). Usage: structure any decision.

Gorry–Scott-Morton grid — decisions by structure (structured/semi/unstructured) × control (operational/managerial/strategic). E.g. order-entry=structured-operational; hiring exec=unstructured-managerial; R&D=unstructured-strategic.

Evolution — 1970s MIS/DSS → 1980s EIS → 1990s OLAP & the term "BI" (coined by Gartner) → 2010s+ AI, mining, Big Data, social.

BI — umbrella term (architectures+tools+DB+apps+methods); "content-free expression". Goal: easy data access so managers act. 4 parts: data warehouse, business analytics, BPM, UI.

OLTP vs OLAP — OLTP = transaction capture (ERP/CRM/SCM/POS), normalized, fast. OLAP = analysis on the warehouse, multidimensional, decision support.

Prescriptive uses descriptive + predictive results to build & pick the best alternative.

Big Data — 3 Vs — Volume, Variety, Velocity; too big/fast for traditional tools; worthless until analysed. Ecosystem = 11 player types (infrastructure, DW, middleware, data-service, descriptive/predictive/prescriptive software, app developers, analysts, academia, regulators, users).

BI applications — customer segmentation, propensity-to-buy, profitability, fraud detection, attrition/churn, channel optimization.

Cases · Sabre & Siemens (Dundas BI dashboards cut cost); specialty steel (available-to-promise dates); CenterPoint Energy (real-time Big Data + sentiment → outage prediction).

Ch.3 · Data, Statistics & Visualization

Key idea · Descriptive analytics = "what happened?"; first know the nature of data (types/scales) and clean it, then summarise (stats) and communicate (viz).

Data taxonomy — Structured (categorical: nominal/ordinal; numerical: interval/ratio) vs Unstructured/semi (text, multimedia, XML/JSON, HTML, logs). Data = lowest abstraction; quality & integrity critical.

Scale eg · jersey #=nominal; rank 1–5=ordinal; °C=interval (0° ≠ none, so 20°≠2×10°); kg=ratio (true 0 → 20kg=2×10kg). Scale picks valid stats/charts.

Preprocessing (4 tasks) — Consolidation (access/select/integrate) → Cleaning (impute mean/median/mode, remove noise/outliers) → Transformation (normalize 0–1, binning, new attrs) → Reduction (PCA/correlation, sampling, balance skew via over/under-sample).

Stats branches — Descriptive (describe data) vs Inferential (infer population from sample). Central tendency: mean/median/mode; dispersion: range/variance/SD/quartiles/box-plot; shape: skewness (asymmetry), kurtosis (peak).

Regression — y=β₀+β₁x (multiple: +β₂x₂…); fit by OLS (min squared error); for explanation & forecasting. Quality: R², p-value, MSE/MAD/RMSE. 5 assumptions: linearity, independence, normality, constant variance, no multicollinearity.

Logistic regression — supervised classification; binary target via sigmoid f(y)=1/(1+e^−(β₀+β₁x)) → probability. Time series models trend+seasonality over time.

Reports — metric mgmt (KPI/SLA), dashboard (many indicators/1 page), balanced-scorecard. Visualization = "insight, not pictures" (Shneiderman). Playfair=line/pie (1801); Minard=Napoleon march (multivariate). Visual analytics = info viz + predictive analytics. Dashboard layers: Monitoring · Analysis · Management (drill-down, show exceptions).

Ch.4 · Data Warehousing & BPM

Key idea · The data warehouse is BI's cleansed, integrated, historical backbone; ETL feeds it, OLAP makes it fast, BPM/scorecards turn analysis into managed action.

Data warehouse — integrated, subject-oriented, time-variant, non-volatile, summarized, NOT normalized, metadata-rich repository for DSS.

Marts/parts — dependent mart (from DW) · independent mart (standalone) · ODS (staging) · EDW (enterprise-wide) · metadata ("data about data").

Architecture — 3-tier (acquisition back-end → DW → client front-end). 5 alternatives: independent marts · data-mart bus (conformed dimensions) · hub-and-spoke · centralized · federated.

3rd option = cloud/hosted DW (low infra). ETL (Extract-Transform-Load) = integration backbone; EAI pushes source→DW; EII real-time multi-source.

Schemas — Star = central fact table + dimension tables (simplest). Snowflake = dimensions normalized into sub-tables. Multidimensionality = dimensions (product/region/time) × measures (revenue/volume) = OLAP cube.

OLAP ops — Slice (1 dim subset) · Dice (multi-dim subset) · Drill down/up (detail↔summary) · Roll-up (aggregate) · Pivot (rotate).

DW vs Data Lake — structured + schema-on-write (SQL) + business users vs raw/native + schema-on-read (NoSQL) + data scientists.

BPM (also CPM/EPM/SEM) closed loop: Strategize→Plan→Monitor/Analyze→Act. KPI: outcome=lagging (revenue), driver=leading (sales leads).

Balanced Scorecard (Kaplan&Norton 1992): Financial · Customer · Internal Process · Learning&Growth around Vision/Strategy. Six Sigma: cut defects (DPMO) via DMAIC (Define-Measure-Analyze-Improve-Control). BSC = strategic management; Six Sigma = performance measurement.

Cases · Maryland (tax fraud via BI+DW); AARP (BI transformation, 347% ROI/3yr); Expedia (customer-satisfaction scorecard); BP Lubricants "BIGS".

Ch.5 · Data Mining (Predictive I)

Key idea · Data mining is a disciplined process (CRISP-DM), not a crystal ball — find valid, useful patterns in structured data: predict (classify/regress) & group (cluster/associate).

Data mining (Fayyad 1996) — nontrivial discovery of valid, novel, useful, understandable patterns. A.k.a. knowledge discovery/extraction, pattern analysis. Blend of stats+ML+AI+DB+viz. Source often a DW. DM ≠ statistics (pattern discovery on large data, by an end user). Patterns: association, prediction, cluster, sequential.

Two contrasts — Classification vs Regression = categorical vs numeric output. Classification vs Clustering = supervised (labelled target) vs unsupervised (no target).

CRISP-DM (6) — Business Understanding → Data Understanding → Data Prep → Model → Test/Eval → Deploy; first 3 ≈ 85% of effort; iterative. Others: SEMMA (SAS: Sample-Explore-Modify-Model-Assess), KDD. Assess models on accuracy, speed, robustness, scalability, interpretability.

Confusion matrix — Accuracy=(TP+TN)/all; Precision=TP/(TP+FP); Recall/TPR=TP/(TP+FN); TNR=TN/(TN+FP).

Eg · TP80,FP20,FN10,TN90 → Acc=170/200=85%, Prec=80/100=80%, Recall=80/90≈89%.

Validation — holdout train⅔/test⅓ (NN 60/20/20); k-fold CV; leave-one-out (k=n); bootstrap (sample w/ replacement); jackknife; AUC/ROC (0.5=chance, 1=perfect, good for skew).

Decision tree — divide-and-conquer; recursively split until each leaf=1 class; differs by splitting / stopping / pruning (pre vs post). Algos: ID3, C4.5, C5, CART, CHAID, M5. Other classifiers: NN, SVM, Bayesian/NB, kNN, GA, rough sets, case-based reasoning.

Ensembles — combine models → robust: bagging (→ random forest), boosting, info fusion.

Clustering — unsupervised, no target, = segmentation. Methods: k-means/k-modes, SOM & ART (neural), fuzzy c-means, GA. k-Means: pick k → random centers → assign nearest → recompute → repeat till stable.

Association rules X→Y [S,C] — Support=P(X∩Y) (how often together); Confidence=P(Y|X) (Y given X). Algos: Apriori (bottom-up frequent itemsets ≥ min support), Eclat, FP-Growth. = market-basket.

Eg · {Laptop, Antivirus}→{Service Plan} [30%,70%]: trio in 30% of baskets; 70% of laptop+AV buyers add the plan → bundle/place together (or apart to force a walk-by).

Myths/mistakes — DM is not instant prediction (it's a deliberate multistep process), needs no dedicated DB or advanced degree, works for any firm size. Top mistake: starting without the end in mind / too little time for data prep.

Cases · Visa (predictive + in-memory analytics cut fraud, keep service); movie financial success (9-class classification, IBM SPSS Modeler); cancer research (ANN vs LR vs RF + variable importance); Miami-Dade Police (predictive policing).

Ch.6 · Text / Web / Social (Predictive II)

Key idea · ~85–90% of corporate data is unstructured text; the trick is to impose structure (term-document matrix) then mine it for topics, sentiment & web/social patterns.

Text analytics = IR + Information Extraction + DM + Web mining (or simply IR + Text Mining). Text mining = semi-auto knowledge extraction from unstructured data. DM vs TM = structured (DB) vs unstructured (Word/PDF/XML); both find novel patterns, semi-automated.

TM application areas — info extraction, topic tracking, summarization, categorization, clustering, concept linking, question answering; domains: law/finance/medicine/marketing; email spam filter.

Terms — corpus/corpora; stemming (running→run); stop words (remove the/is/and); tokenizing; POS tagging; morphology; TDM (term-by-document matrix); SVD/LSI (reduce sparse matrix, like PCA).

NLP — bag-of-words → "understanding" (AI + comp. linguistics). Challenges: POS tagging, text segmentation, word-sense disambiguation, syntactic ambiguity. Resources: WordNet, SentiWordNet.

Text-mining 3 steps — 1 Establish corpus (collect/digitize) → 2 Create TDM (impose structure; cells = counts/binary/log/TF-IDF; reduce via SVD) → 3 Extract knowledge (classify/cluster/associate/trend).

Topic modeling — unsupervised theme discovery: freq clustering, LSI/LSA (TF-IDF+SVD), LDA (most popular: doc=mix of topics, topic=word distribution).

Sentiment analysis — "what do people feel?"; apps: VOC/VOM/VOE, brand, financial, politics, gov intelligence. 4 steps: (1) detect objectivity S–O (fact vs opinion) → (2) N–P polarity (neg/pos) → (3) target identification → (4) aggregation (word→stmt→para→doc).

Eg · "screen gorgeous but battery terrible" → screen=P, battery=N (two targets).

SNA = graph of actors; metrics: centrality, density, betweenness/bridge, structural holes, cliques, clustering coefficient, cohesion. Social media analytics = Web-2.0 content → competitive insight; ladder Inactives→Spectators→Joiners→Collectors→Critics→Creators.

Cases · "Mining for Lies" deception detection (371 statements, 31 cues → LR 67.3%, DT 71.6%, NN 73.5%); AMC Networks (predict ratings); Lenovo (sentiment+warranty+call-center redesign); IS literature (901 abstracts → 9 clusters); Tito's Vodka (social brand loyalty).

Formulas, Metrics & Quick-Pick

Classification — Accuracy=(TP+TN)/(TP+TN+FP+FN); Precision=TP/(TP+FP); Recall/TPR=TP/(TP+FN); TNR=TN/(TN+FP); F1=2·P·R/(P+R) (best for imbalance); AUC 0.5=random, 1=perfect.

Regression — y=β₀+β₁x; fit=OLS; quality=R², p-value, RMSE/MAD/MSE. Logistic 1/(1+e^−(β₀+β₁x)) → P(class). Tree split = Gini / info-gain(entropy); ensembles = bagging(→RF) / boosting.

Association — Support=P(X∩Y); Confidence=P(Y|X); Lift=Conf/P(Y); >1 positive, =1 independent, <1 negative. Clustering quality — Silhouette ↑ better, Davies-Bouldin ↓ better.

Text — TF-IDF=TF×log(N/dfₜ) (high = frequent here, rare overall); reduce TDM via SVD. DPMO=defects/(units×opportunities)×10⁶; Six Sigma=3.4 DPMO.

Which technique? — categorical target+labels → classification (tree/NN/SVM/NB); numeric target → regression; no target, find groups → clustering (k-means); items bought together → association (Apriori); unstructured text → text mining/LDA/sentiment; "what happened?" → descriptive/viz; "best action?" → prescriptive/optimization.
