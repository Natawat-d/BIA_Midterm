Ch.1 ¬∑ Overview of BI & Analytics

Key idea ¬∑ BI is the modern evolution of DSS; its job is to turn data into managerial action, and every method is descriptive, predictive or prescriptive.

DSS ‚Äî interactive computer system that uses data + models to solve semi/unstructured problems (Gorry & Scott-Morton 1971). 4 components: data mgmt ¬∑ model mgmt ¬∑ knowledge subsystem ¬∑ UI. Usage: couple human judgment + computing to raise decision quality.

Simon's 4 phases ‚Äî Intelligence (find problem) ‚Üí Design (build model/options) ‚Üí Choice (pick) ‚Üí Implementation (act+monitor). Usage: structure any decision.

Gorry‚ÄìScott-Morton grid ‚Äî decisions by structure (structured/semi/unstructured) √ó control (operational/managerial/strategic). E.g. order-entry=structured-operational; hiring exec=unstructured-managerial; R&D=unstructured-strategic.

Evolution ‚Äî 1970s MIS/DSS ‚Üí 1980s EIS ‚Üí 1990s OLAP & the term "BI" (coined by Gartner) ‚Üí 2010s+ AI, mining, Big Data, social.

BI ‚Äî umbrella term (architectures+tools+DB+apps+methods); "content-free expression". Goal: easy data access so managers act. 4 parts: data warehouse, business analytics, BPM, UI.

OLTP vs OLAP ‚Äî OLTP = transaction capture (ERP/CRM/SCM/POS), normalized, fast. OLAP = analysis on the warehouse, multidimensional, decision support.

Big Data ‚Äî 3 Vs ‚Äî Volume, Variety, Velocity; too big/fast for traditional tools; worthless until analysed. Ecosystem = 11 player types (infrastructure, DW, middleware, data-service, descriptive/predictive/prescriptive software, app developers, analysts, academia, regulators, users).

BI applications ‚Äî customer segmentation, propensity-to-buy, profitability, fraud detection, attrition/churn, channel optimization.

Data taxonomy ‚Äî Structured (categorical: nominal/ordinal; numerical: interval/ratio) vs Unstructured/semi (text, multimedia, XML/JSON, HTML, logs). Data = lowest abstraction; quality & integrity critical.

Ch.4 ¬∑ Data Warehousing & BPM

Key idea ¬∑ The data warehouse is BI's cleansed, integrated, historical backbone; ETL feeds it, OLAP makes it fast, BPM/scorecards turn analysis into managed action.

Data warehouse ‚Äî integrated, subject-oriented, time-variant, non-volatile, summarized, NOT normalized, metadata-rich repository for DSS.

Marts/parts ‚Äî dependent mart (from DW) ¬∑ independent mart (standalone) ¬∑ ODS (staging) ¬∑ EDW (enterprise-wide) ¬∑ metadata ("data about data").

Architecture ‚Äî 3-tier (acquisition back-end ‚Üí DW ‚Üí client front-end). 5 alternatives: independent marts ¬∑ data-mart bus (conformed dimensions) ¬∑ hub-and-spoke ¬∑ centralized ¬∑ federated.

Ch.5 ¬∑ Data Mining (Predictive I)

Key idea ¬∑ Data mining is a disciplined process (CRISP-DM), not a crystal ball ‚Äî find valid, useful patterns in structured data: predict (classify/regress) & group (cluster/associate).

Data mining (Fayyad 1996) ‚Äî nontrivial discovery of valid, novel, useful, understandable patterns. A.k.a. knowledge discovery/extraction, pattern analysis. Blend of stats+ML+AI+DB+viz. Source often a DW. DM ‚âÝ statistics (pattern discovery on large data, by an end user). Patterns: association, prediction, cluster, sequential.

Two contrasts ‚Äî Classification vs Regression = categorical vs numeric output. Classification vs Clustering = supervised (labelled target) vs unsupervised (no target).

CRISP-DM (6) ‚Äî Business Understanding ‚Üí Data Understanding ‚Üí Data Prep ‚Üí Model ‚Üí Test/Eval ‚Üí Deploy; first 3 ‚âà 85% of effort; iterative. Others: SEMMA (SAS: Sample-Explore-Modify-Model-Assess), KDD. Assess models on accuracy, speed, robustness, scalability, interpretability.

Confusion matrix ‚Äî Accuracy=(TP+TN)/all; Precision=TP/(TP+FP); Recall/TPR=TP/(TP+FN); TNR=TN/(TN+FP).

Eg ¬∑ TP80,FP20,FN10,TN90 ‚Üí Acc=170/200=85%, Prec=80/100=80%, Recall=80/90‚âà89%.

Validation ‚Äî holdout train‚Öî/test‚Öì (NN 60/20/20); k-fold CV; leave-one-out (k=n); bootstrap (sample w/ replacement); jackknife; AUC/ROC (0.5=chance, 1=perfect, good for skew).

Decision tree ‚Äî divide-and-conquer; recursively split until each leaf=1 class; differs by splitting / stopping / pruning (pre vs post). Algos: ID3, C4.5, C5, CART, CHAID, M5. Other classifiers: NN, SVM, Bayesian/NB, kNN, GA, rough sets, case-based reasoning.

Ensembles ‚Äî combine models ‚Üí robust: bagging (‚Üí random forest), boosting, info fusion.

Clustering ‚Äî unsupervised, no target, = segmentation. Methods: k-means/k-modes, SOM & ART (neural), fuzzy c-means, GA. k-Means: pick k ‚Üí random centers ‚Üí assign nearest ‚Üí recompute ‚Üí repeat till stable.

Association rules X‚ÜíY [S,C] ‚Äî Support=P(X‚à©Y) (how often together); Confidence=P(Y|X) (Y given X). Algos: Apriori (bottom-up frequent itemsets ‚â• min support), Eclat, FP-Growth. = market-basket.

Eg ¬∑ {Laptop, Antivirus}‚Üí{Service Plan} [30%,70%]: trio in 30% of baskets; 70% of laptop+AV buyers add the plan ‚Üí bundle/place together (or apart to force a walk-by).

Myths/mistakes ‚Äî DM is not instant prediction (it's a deliberate multistep process), needs no dedicated DB or advanced degree, works for any firm size. Top mistake: starting without the end in mind / too little time for data prep.

Cases ¬∑ Visa (predictive + in-memory analytics cut fraud, keep service); movie financial success (9-class classification, IBM SPSS Modeler); cancer research (ANN vs LR vs RF + variable importance); Miami-Dade Police (predictive policing).

Ch.6 ¬∑ Text / Web / Social (Predictive II)

Key idea ¬∑ ~85‚Äì90% of corporate data is unstructured text; the trick is to impose structure (term-document matrix) then mine it for topics, sentiment & web/social patterns.

Text analytics = IR + Information Extraction + DM + Web mining (or simply IR + Text Mining). Text mining = semi-auto knowledge extraction from unstructured data. DM vs TM = structured (DB) vs unstructured (Word/PDF/XML); both find novel patterns, semi-automated.

TM application areas ‚Äî info extraction, topic tracking, summarization, categorization, clustering, concept linking, question answering; domains: law/finance/medicine/marketing; email spam filter.

Terms ‚Äî corpus/corpora; stemming (running‚Üírun); stop words (remove the/is/and); tokenizing; POS tagging; morphology; TDM (term-by-document matrix); SVD/LSI (reduce sparse matrix, like PCA).

NLP ‚Äî bag-of-words ‚Üí "understanding" (AI + comp. linguistics). Challenges: POS tagging, text segmentation, word-sense disambiguation, syntactic ambiguity. Resources: WordNet, SentiWordNet.

Text-mining 3 steps ‚Äî 1 Establish corpus (collect/digitize) ‚Üí 2 Create TDM (impose structure; cells = counts/binary/log/TF-IDF; reduce via SVD) ‚Üí 3 Extract knowledge (classify/cluster/associate/trend).

Topic modeling ‚Äî unsupervised theme discovery: freq clustering, LSI/LSA (TF-IDF+SVD), LDA (most popular: doc=mix of topics, topic=word distribution).

Sentiment analysis ‚Äî "what do people feel?"; apps: VOC/VOM/VOE, brand, financial, politics, gov intelligence. 4 steps: (1) detect objectivity S‚ÄìO (fact vs opinion) ‚Üí (2) N‚ÄìP polarity (neg/pos) ‚Üí (3) target identification ‚Üí (4) aggregation (word‚Üístmt‚Üípara‚Üídoc).

Eg ¬∑ "screen gorgeous but battery terrible" ‚Üí screen=P, battery=N (two targets).

Formulas, Metrics & Quick-Pick

Classification ‚Äî Accuracy=(TP+TN)/(TP+TN+FP+FN); Precision=TP/(TP+FP); Recall/TPR=TP/(TP+FN); TNR=TN/(TN+FP); F1=2¬∑P¬∑R/(P+R) (best for imbalance); AUC 0.5=random, 1=perfect.

Regression ‚Äî y=Œ≤‚ÇÄ+Œ≤‚ÇÅx; fit=OLS; quality=R¬≤, p-value, RMSE/MAD/MSE. Logistic 1/(1+e^‚àí(Œ≤‚ÇÄ+Œ≤‚ÇÅx)) ‚Üí P(class). Tree split = Gini / info-gain(entropy); ensembles = bagging(‚ÜíRF) / boosting.

Association ‚Äî Support=P(X‚à©Y); Confidence=P(Y|X); Lift=Conf/P(Y); >1 positive, =1 independent, <1 negative. Clustering quality ‚Äî Silhouette ‚Üë better, Davies-Bouldin ‚Üì better.

Text ‚Äî TF-IDF=TF√ólog(N/df‚Çú) (high = frequent here, rare overall); reduce TDM via SVD. DPMO=defects/(units√óopportunities)√ó10‚Å∂; Six Sigma=3.4 DPMO.

Which technique? ‚Äî categorical target+labels ‚Üí classification (tree/NN/SVM/NB); numeric target ‚Üí regression; no target, find groups ‚Üí clustering (k-means); items bought together ‚Üí association (Apriori); unstructured text ‚Üí text mining/LDA/sentiment; "what happened?" ‚Üí descriptive/viz; "best action?" ‚Üí prescriptive/optimization.
