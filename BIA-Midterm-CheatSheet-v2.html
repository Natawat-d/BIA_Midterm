Ch.1 · Overview of BI, Analytics & Data Science

Key idea · BI (Business Intelligence) is the modern evolution of DSS; its job is to turn data into managerial action, and every method is descriptive, predictive or prescriptive.

Need for DSS (Decision Support System) — faster computing + big-data processing (hardware/software/network growth); collaboration (share dashboards & reports); better data management (databases + data warehouses organise enterprise data); big data (large structured + unstructured datasets); analytical tools (dashboards, OLAP, data mining); human cognitive limits (let systems analyse large data); knowledge management (store organisational experience); anywhere support (cloud + mobile). Purpose: transform data → information → insight → decision.

Relationship — DSS = early decision-support systems; BI = evolution of DSS with better tools/analytics; BA (Business Analytics) = the analytical techniques used within BI.

DSS — interactive computer system using data + models to solve semi/unstructured problems (Gorry & Scott-Morton 1971); combines human judgment + computing to raise decision quality. 4 components: data management · model management · knowledge-based subsystem (org knowledge/rules) · user interface (UI) — plus the user.

Simon's phases (Simon 1977) — Intelligence (identify problem/opportunity, gather info) → Design (build options/models, analyse alternatives) → Choice (evaluate & select best) → Implementation (execute) → Monitoring (review, adjust). Car eg: car too old → compare sedan/SUV/EV → pick affordable+efficient → buy → track fuel/maintenance.

Gorry–Scott-Morton grid — decision structure (structured/semi/unstructured) × control (operational/managerial/strategic). Structured-operational = order entry, payroll; semi = credit approval, scheduling; unstructured-strategic = mergers (M&A), R&D. Operational→more structured; strategic→more unstructured.

BI — umbrella / content-free term (architectures + databases + tools + analytical methods + applications). Goal: easy data access so managers analyse & decide. Process Data→Information→Knowledge→Decisions→Actions. 4 components: data warehouse · business analytics · business performance management (BPM) · UI (future = intelligent systems). Architecture flow: data sources → DW (Data Warehouse) → business analytics → performance management → UI.

OLTP vs OLAP — OLTP (Online Transaction Processing) = operational databases (ERP / SCM / CRM / POS), goal = data capture, normalized, fast. OLAP (Online Analytical Processing) = runs on the DW, goal = decision support, multidimensional. (ERP=Enterprise Resource Planning, SCM=Supply Chain Management, CRM=Customer Relationship Management, POS=Point of Sale.)

Big Data — 3 Vs: Volume (size, TB→PB→ZB), Velocity (real-time generation/flow), Variety (structured + unstructured); some add Veracity (trust/quality) + Value. Too big/fast for traditional tools — only creates value when analysed. Enabling tech: Hadoop (HDFS + MapReduce), NoSQL, Spark, streaming, cloud.

Analytics ecosystem (10 players) — data-generation providers (sensors / IoT = Internet of Things); data-management providers (databases / cloud); data-warehouse providers; middleware providers (integration); analytics-software developers (BI tools); application developers (industry apps); data-service providers; academic institutions (education/certification); regulators (data governance); user organisations.

Ch.3 · Descriptive Analytics I — Data, Statistics & Visualization

Key idea · Descriptive analytics = "what happened?"; first know the nature of data (types/scales) and clean it, then summarise (statistics) and communicate (visualization).

Nature of data — facts from observation/experiment (numbers, text, images); lowest abstraction → information → knowledge → decisions; quality & integrity are critical. Analytics-ready quality metrics: source reliability, content accuracy, accessibility, security/privacy, richness, consistency, timeliness (currency), granularity (detail), validity/relevance.

Scales of measurement — Nominal (categories: gender, country) · Ordinal (ordered: rankings) · Interval (equal gaps, no true zero: °C) · Ratio (equal gaps + true zero: income, age). Scale decides which stats & charts are valid.

Preprocessing (4 tasks) — Consolidation (access/select/integrate sources) → Cleaning (impute mean/median/mode, remove noise/outliers) → Transformation (normalize 0–1, binning, new attributes) → Reduction (PCA = Principal Component Analysis, correlation, sampling, balance skew via over/under-sampling).

Statistics — Descriptive (describe data) vs Inferential (infer population from sample). Central tendency: mean/median/mode. Dispersion: range (max−min), variance, standard deviation (SD), quartiles, box-plot. Shape: histogram, skewness (asymmetry), kurtosis (peak/tail).

Regression — model input x → output y; y=β₀+β₁x; fit by OLS (Ordinary Least Squares); for explanation + forecasting (part of inferential stats). Steps: assess → fit → evaluate → deploy. Quality: R² (variance explained), p-value, error MSE/MAD/RMSE. 5 assumptions: linearity, independence, normality, constant variance, no multicollinearity. Correlation = strength/direction only; regression also predicts + gives predictor impact. Logistic regression = classification via sigmoid → probability.

Business reporting — a report = communication artifact for decisions; sources internal + external (via ETL). Functions: operational control, inform, present analysis, persuade, organizational memory. Types: metric-management (KPI = Key Performance Indicator / SLA = Service Level Agreement), dashboard (graphical summary, one page), balanced-scorecard (financial/customer/internal/learning).

Visualization = "insight, not pictures" (Shneiderman). Data viz = raw representation; information viz = summarized/contextual (storytelling). Visual analytics = information viz + predictive analytics (what happened + what will). Playfair (line/pie, 1801); Minard (Napoleon's march, multivariate). Advanced: heat map, tree map, Sankey, bubble, radar, candlestick.

Ch.4 · Descriptive Analytics II — Data Warehousing & BPM

Key idea · The data warehouse (DW) is BI's cleansed, integrated, historical backbone; ETL feeds it, OLAP makes it fast, BPM (Business Performance Management) / scorecards turn analysis into managed action.

Data warehouse — subject-oriented, integrated, time-variant, non-volatile, summarized, NOT normalized, metadata-rich repository for DSS. Data marts = departmental DW subset: dependent (built from the enterprise DW) / independent (standalone). ODS (Operational Data Store) = temporary staging area; EDW (Enterprise Data Warehouse) = org-wide; metadata = "data about data" (structure/origin/usage; supports governance).

Framework & architecture — sources → ETL → EDW → data marts → analytics (OLAP/dashboards/mining/reporting). 3-tier (Tier 1 client UI → Tier 2 application/OLAP server → Tier 3 database server) vs 2-tier (combined; simpler, less scalable). 5 alternatives: independent marts · data-mart bus (conformed dimensions) · hub-and-spoke · centralized · federated. Selection factors: info-sharing needs, mgmt requirements, urgency, end-user tasks, resources, strategic vision, IT compatibility, staff skills, complexity, politics.

Integration — ETL (Extract-Transform-Load) = integration backbone; EAI (Enterprise Application Integration) pushes source→DW; EII (Enterprise Information Integration) = real-time multi-source view. Development: Inmon top-down (build EDW first) vs Kimball bottom-up (build marts first, integrate later).

Dimensional modeling — fact tables (numeric measures) + dimension tables. Star = central fact + dimensions (simplest, fast). Snowflake = dimensions normalized into sub-tables (less redundancy, more complex). Multidimensional = dims (product/region/time/salesperson) × measures (revenue/profit) = OLAP cube.

OLAP operations — Slice (1 dimension) · Dice (multi-dim subset) · Drill down/up (detail↔summary) · Roll-up (aggregate) · Pivot (rotate). DW vs Data Lake — structured + schema-on-write + business analysts vs raw/native + schema-on-read + data scientists (big data, cheaper storage).

BPM closed loop: Strategize → Plan → Monitor/Analyze → Act; gives real-time alerts on risks/opportunities. Performance measurement = compare actual vs planned. KPI: outcome = lagging indicator (revenue), driver = leading indicator (sales leads); needs targets + benchmarks + timeframe + strategic alignment.

Balanced Scorecard (BSC) (Kaplan&Norton 1992): translates strategy into 4 linked perspectives — Financial · Customer · Internal Process · Learning&Growth — around Vision/Strategy. Six Sigma: cut defects via DMAIC (Define-Measure-Analyze-Improve-Control) toward 3.4 DPMO (Defects Per Million Opportunities). BSC = strategic management framework; Six Sigma = process-improvement methodology. Effective measurement: focus on key success factors; past/present/future; balance stakeholders; align top→ops; realistic targets.

Ch.5 · Predictive Analytics I — Data Mining

Key idea · Data mining (DM) is a disciplined process (CRISP-DM), not a crystal ball — find valid, novel, useful, understandable patterns in structured data: predict (classify/regress) & group (cluster/associate).

Data mining (Fayyad 1996) — nontrivial discovery of patterns; a.k.a. knowledge discovery/extraction. Blend of statistics + ML (Machine Learning) + AI (Artificial Intelligence) + databases + visualization; source often a DW; end-users mine. DM ≠ statistics (pattern discovery on large data, by an end user). Why now: competition, data value, big data, data warehouses, cheap storage/compute, data-driven culture.

How DM works — finds patterns: association · prediction · cluster · sequential/time-series. 8 tasks: prediction, classification, regression, clustering, association, link analysis, sequence analysis, outlier/anomaly. Applications: CRM (churn, cross-sell), banking (fraud, credit risk), retail (inventory, placement), manufacturing (predictive maintenance), insurance, stock forecasting, healthcare diagnosis, crime/government.

Two contrasts — Classification vs Regression = categorical vs numeric output. Classification vs Clustering = supervised (labelled target) vs unsupervised (no target). Market-basket analysis = items bought together (diaper+beer) → layout/cross-sell/promotion. (Algos above: DT=Decision Tree, ANN=Artificial Neural Network, SVM=Support Vector Machine, kNN=k-Nearest Neighbor, NB=Naïve Bayes, SOM=Self-Organizing Map.)

Processes — CRISP-DM (Cross-Industry Standard Process for DM, 6 phases): Business Understanding → Data Understanding → Data Prep → Modeling → Evaluation → Deployment; prep ≈ 85% effort, iterative. SEMMA (Sample-Explore-Modify-Model-Assess, by SAS). KDD (Knowledge Discovery in Databases): select → clean → transform → mine → interpret.

Classification = supervised, predicts categorical class (spam/credit/diagnosis). Confusion matrix — TP/FP/TN/FN; Accuracy=(TP+TN)/all; Precision=TP/(TP+FP); Recall/TPR (True Positive Rate)=TP/(TP+FN); TNR=TN/(TN+FP). Validation: holdout 70/30; k-fold CV (Cross-Validation); leave-one-out; bootstrap. ROC (Receiver Operating Characteristic) / AUC (Area Under Curve): 0.5=chance, 1=perfect (good for skew).

Eg · TP80, FP20, FN10, TN90 → Accuracy 85%, Precision 80%, Recall ≈89%.

Decision tree — divide-and-conquer, recursively split until pure leaf; differ by split / stop / prune. Algorithms: ID3, C4.5, CART, CHAID. Other classifiers: NN, SVM, NB, kNN, GA (Genetic Algorithm), CBR (Case-Based Reasoning). Ensembles (combine models → robust): bagging (→ Random Forest), boosting, info fusion.

Clustering — unsupervised segmentation (no target). k-Means: pick k → random centers → assign nearest → recompute centers → repeat till stable. Also hierarchical, SOM, fuzzy c-means, genetic.

Association rules X→Y [S,C] — Support=P(X∩Y) (how often together); Confidence=P(Y|X) (Y given X); Lift=Conf/P(Y) (>1 positive). Apriori = bottom-up frequent itemsets ≥ min support; also Eclat, FP-Growth. Myths/mistakes: DM is not instant prediction (deliberate multistep process); top mistake = no end-in-mind / too little time on data prep.

Ch.6 · Predictive Analytics II — Text, Web & Social Media

Key idea · ~85–90% of corporate data is unstructured text; impose structure (term-document matrix) then mine it for topics, sentiment & web/social patterns.

Text analytics = IR (Information Retrieval) + Information Extraction + DM + Web mining (≈ IR + Text Mining). DM vs TM (Text Mining) = structured data (databases) vs unstructured (Word/PDF/web/email); both are semi-automated pattern discovery, but TM must impose structure first. TM applications: information extraction, topic tracking, summarization, categorization, clustering, concept linking, question answering; spam filtering.

Terms — corpus (document collection); tokenizing (split into words); stop words (remove the/is/and); stemming (running→run); POS (Part-Of-Speech) tagging; TDM (Term-Document Matrix). NLP (Natural Language Processing) = structure/"understand" text (AI + linguistics); challenges: word-sense disambiguation, syntactic ambiguity, text segmentation. Resource: WordNet (lexical database).

Text mining — 3 steps: 1 Establish corpus (collect/digitize) → 2 Create TDM (cells = counts / binary / TF-IDF = Term Frequency–Inverse Document Frequency; reduce sparse matrix via SVD = Singular Value Decomposition ≈ PCA) → 3 Extract knowledge (classify / cluster / associate / trend).

Topic modeling — unsupervised theme discovery: word-frequency clustering, LSI/LSA (Latent Semantic Indexing/Analysis = TF-IDF + SVD), LDA (Latent Dirichlet Allocation — most popular: a document = mix of topics, a topic = word distribution).

Sentiment analysis — "what do people feel?". 4 steps: (1) S–O detection (objective fact vs opinion) → (2) N–P polarity (negative/positive) → (3) target identification → (4) aggregation (word→statement→doc). Apps: VOC / VOM / VOE (Voice of Customer / Market / Employee), brand, financial, politics, government intelligence.

Eg · "screen gorgeous but battery terrible" → screen=Positive, battery=Negative (two targets).

↔Ch.1R1 · A Taxonomy of Decision Support Systems (Alter 1977)

  • Argument: DSS is not homogeneous — systems differ widely in what they do and how directly they shape the decision.
  • Problem: DSS were ad-hoc; no shared language to compare, design or communicate them.
  • Method: studied 56 real systems → ranked by degree of action implication (output just informs vs directly determines the decision) → placed on a data-oriented ↔ model-oriented spectrum → named 7 types.
  • 7 types: data-oriented — File Drawer (direct data access), Data Analysis (ad-hoc manipulation), Analysis Information (decision DB + small models); model-oriented — Accounting (definitional calc, budgets), Representational (simulate/estimate, approximate models), Optimization (best solution under constraints), Suggestion (recommends a specific action).
  • Compare: data-oriented used hands-on by decision-makers; model-oriented often need staff intermediaries. Accounting models trusted (definitional); representational doubted (approximations).
  • 5 lessons: implementation challenge depends on type • user role shifts across the taxonomy • model credibility varies • taxonomy = design/communication tool • approaches can be combined.
  • One-liner: Alter's 7-type taxonomy (by action implication, data→model) gives a language to design, classify & adopt DSS.

Formulas, Metrics & Quick-Pick

Classification — Accuracy=(TP+TN)/all; Precision=TP/(TP+FP); Recall/TPR=TP/(TP+FN); TNR=TN/(TN+FP); F1=2·P·R/(P+R) (best for imbalance); AUC 0.5=random, 1=perfect.

Regression — y=β₀+β₁x; fit=OLS; quality=R², p-value, RMSE/MAD/MSE. Logistic 1/(1+e^−(β₀+β₁x)) → P(class). Tree split = Gini / info-gain(entropy); ensembles = bagging(→RF) / boosting.

Association — Support=P(X∩Y); Confidence=P(Y|X); Lift=Conf/P(Y) (>1 positive, =1 independent, <1 negative). Clustering — Silhouette ↑ better, Davies-Bouldin ↓ better.

Text — TF-IDF=TF×log(N/dfₜ) (high = frequent here, rare overall); reduce TDM via SVD. DPMO=defects/(units×opportunities)×10⁶; Six Sigma=3.4 DPMO.

Which technique? — categorical target+labels → classification (tree/NN/SVM/NB); numeric target → regression; no target, find groups → clustering (k-means); items bought together → association (Apriori); unstructured text → text mining/LDA/sentiment; "what happened?" → descriptive/viz; "best action?" → prescriptive/optimization.
