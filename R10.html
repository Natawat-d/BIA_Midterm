Text-Based Content Analysis on Social Media Using Topic Modelling to Support Digital Marketing

Gandhi Surya Buana, Raras Tyasnurita, Nindita Cahya Puspita, Retno Aulia Vinarti & Faizal Mahananto, Institut Teknologi Sepuluh Nopember (ITS), Surabaya, Indonesia · JOIV, 8(1), Mar 2024, pp. 88–95.

One-sentence takeaway: no existing SMA tool discovers topics for text-in-image Instagram content, so the authors build one that scrapes posts, uses OCR to pull text out of images and LDA to find dominant topics — helping marketers choose content ideas (validated at 93% by experts).

Problem & aim

Instagram is huge in Indonesia (98M+ users, 2021) and text-based content is trending because audiences distrust traditional marketing. Content creators need an ideation process — choosing a relevant topic. Existing SMA tools (Instagram Insight, SproutSocial, Brandwatch) use metadata/captions/comments but none analyse the text rendered on the image. Aim: build an SMA tool that uses OCR + topic modelling to discover topic references from text-based Instagram content, supporting digital-marketing strategy.

Key concepts & definitions

Social Media Analytics (SMA)
Tools/systems that turn social content into insight (dashboards, listening, sentiment, topics).
OCR (Optical Character Recognition)
Converts an image's text into machine-readable characters — image input → pre-processing → segmentation → normalization → extraction → recognition.
Topic modelling / LDA
Latent Dirichlet Allocation — unsupervised modelling that identifies latent topics in a document collection; each document = a mix of topics, each topic = a word distribution.
Coherence score
A measure used to choose the optimal number of topics (the peak coherence).

Data & the retrieval pipeline

Data was scraped from the dibimbing.id Instagram account (2023-01-05 → 2023-04-06) using the Instaloader Python library — collecting likes, comments, captions, upload times and content images (.jpg) + metadata (.json). Pipeline (Fig. 1): scrape target → extract text from images (OCR) + metadata → store in a data frame (CSV). Caption text and image text were combined (same context) to save memory.

Method — three software classes

ClassJob
InstaScrapperCollect Instagram data (images + metadata) for the target account.
CleanPre-processing: tokenizing → filtering (remove unimportant tokens / stop-words, English + Indonesian) → stemming → remove empty text → build corpus/dictionary.
TopicModellingRun LDA; pick the topic count by the highest coherence score; label each post with its dominant topic; visualise.

Visualizations used: word cloud + bar chart (topics), scatter plot (content distribution over a period), bar chart (distribution by hour/day).

Results

OCR performance

Across 6 content images, OCR extracted 210 of 255 words = 82.3% accuracy (range 44–97% per image). Misses were brand logos ("dibimbing.id"), graphic text and low contrast.

Topic modelling

The coherence score peaked at 4 topics (optimal). The four discovered topics:

  • Topic 0 — UI/UX design & website matters.
  • Topic 1 — SEO, website data, Google-related.
  • Topic 2 — job listings & portfolio.
  • Topic 3 — career, classes & skills (web development, data science, business).

Content-performance insight: comments & likes drive topic engagement (virality), separate from sheer upload count. Best upload windows: Wed/Thu/Sat, 17:00–18:00 WIB.

Expert evaluation

  • Understanding & usage test — of 15 questions, the average participant answered 14 correct (93%) → testers could use the SMA to find topic references.
  • User acceptance (SUS, scale 1–5) — average ≈ 85% acceptance; lowest scores came from non-IT users needing programming help.

Conclusions & limitations

The SMA tool was built with three classes (InstaScrapper, Clean, TopicModelling); success measured by OCR accuracy (82.3%) and LDA topic recommendations; it also yields content-performance & upload-strategy insight. It is useful but needs UX refinement.

Limitations / next steps: the tool requires programming proficiency (a barrier for non-IT marketers); OCR struggles with logos/low contrast. Future work: improve the user experience (a web app) and target users such as Data Analysts / BI Analysts.
