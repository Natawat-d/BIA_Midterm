One-sentence takeaway: to sharpen marketing, the study segments fashion-transaction data with K-Means vs DBSCAN (CRISP-DM; PCA + Standard Scaler), evaluating by Silhouette Score & Davies-Bouldin Index — and finds K-Means superior at k = 7 (Silhouette 0.549, DBI 0.593) over DBSCAN (0.29 / 0.92), then deploys it as a web dashboard.

Problem & aim

Indonesian e-commerce is booming (fashion = ~70% of online transactions in 2023). The "XYZ" fashion company wants to understand consumer behaviour to improve marketing & respond to needs, increasing sales. The study builds a customer segmentation model using clustering (K-Means & DBSCAN) on Shopee transaction data to identify customer groups by purchasing pattern, providing a basis for targeted, optimal marketing.

Key concepts & definitions

K-Means
Centroid-based partitional clustering — assigns points to the nearest of k centroids, recomputes; you must set k. Objective: minimise within-cluster squared distance J = Σ Σ ‖xᵢ − μk‖².
DBSCAN
Density-Based Spatial Clustering of Applications with Noise — finds clusters as dense regions; points in sparse regions are noise. Two parameters: ε (epsilon) = max neighbour distance, minPts = min points for a core. Handles arbitrary shapes & outliers; no preset k.
Standard Scaler
Standardises features z = (x−μ)/σ so scale doesn't dominate distance-based clustering.
PCA
Principal Component Analysis — reduces dimensionality by projecting to fewer components, removing redundant columns for cleaner clustering.

Data & the CRISP-DM workflow

Data = 10,845 rows × 49 columns of Shopee sales (Aug 2023–Mar 2024), gathered by purposive sampling and scraping the Shopee Seller Center, exported to CSV. The study follows the six CRISP-DM phases:

  1. Business Understanding — XYZ wants to optimise marketing & find improvement areas.
  2. Data Understanding — monthly sales volumes (e.g. Nov 2023 = 1,750; Dec = 1,597 rows).
  3. Data Preparation — clean NaNs, encode categoricals, Standard Scaler + PCA.
  4. Modeling — K-Means (find optimal k) & DBSCAN (tune ε, minPts).
  5. Evaluation — Silhouette Score & Davies-Bouldin Index.
  6. Deployment — a web app (FastAPI + React/TypeScript).

Preprocessing details

  • Cleaning — remove NaN values; a correlation-matrix heatmap shows e.g. Initial Price, Price After Discount & Total Product Price correlate ~0.74 with total payment.
  • Label encoding — convert categorical (object) columns to numeric.
  • Standard Scaler — equalise feature contribution.
  • PCA — reduce to principal components (2 for visualisation) to overcome high dimensionality.

Evaluation metrics (no labels exist → use cluster validity)

MetricRange / directionMeaning
Silhouette Score−1 … +1, higher = betterHow well each point fits its cluster vs others s = (b−a)/max(a,b).
Davies-Bouldin Index (DBI)≥ 0, lower = betterAverage inter-cluster similarity (homogeneity); lower = cleaner partition.

(They also mention the Calinski-Harabasz Index, which agreed with Silhouette on the best DBSCAN ε/minPts.) Accuracy/precision cannot be used — there is no ground-truth label.

Both Silhouette and DBI agree the optimal K-Means cluster count is k = 7. K-Means won (higher Silhouette, lower DBI) because DBSCAN struggled with the data's varying density. The 7 clusters were profiled by their top-7 best-selling products (pie charts per cluster) to characterise each segment.

Deployment

The model became a web application: FastAPI (Python) backend runs the data-processing + ML pipeline; React + TypeScript frontend lets a user upload a CSV, run the model, and view output tables + per-cluster pie charts of best-selling products — turning analytics into an operational marketing tool.

Conclusions & limitations

Clustering on fashion-transaction data with the CRISP-DM framework, preprocessed by Standard Scaler + PCA and evaluated by Silhouette + DBI, shows K-Means outperforms DBSCAN (0.549/0.593 vs 0.29/0.92) at k = 7; the model was deployed as a web app to support segmentation decisions.

Limitations / next steps: the dataset may not be fully representative; updating with more recent data could improve accuracy; future work should explore Gaussian Mixture Models (GMM) and more robust real-world deployment.
